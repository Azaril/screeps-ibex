# Engineering Practices — the review reference

> **Purpose.** The cross-cutting engineering rules this project holds changes to, in one citable place. Use it during code/design review: each rule has a stable ID (`EP-<section>.<n>`), a one-line statement, the failure it prevents (the real incident where one exists), and the authoritative source. Cite IDs in review comments.
>
> **Relationship to other docs.** [`AGENTS.md`](../../AGENTS.md) is codebase orientation (domain, layout, tick flow) plus its own conventions (§8) — this doc extends it and wins on conflict (AGENTS.md §6/§9 carry pointers here where they went stale). ADRs own *decisions*; this doc owns *practices* — a rule that names an owned registry (shed tiers, segments) defers to the owning ADR. Synthesized 2026-06-11 from the review corpus, ADRs 0001–0015, Phase 0/1 execution, and operator directives; provenance cited per rule.
>
> **Lifecycle.** Living document. New rules come from operator directives or incident postmortems and cite their origin; rules nobody can trace get deleted. Changes land like any other doc change (commit on master, reviewable diff).

## 0. Quick review checklist

The highest-frequency catches. Full rules below.

- [ ] New mutable `static`/`thread_local!`? Rejected unless it's one of the EP-1.1 sanctioned exceptions (composition root, platform trio, log-once latch, feature-gated profiling) (§1).
- [ ] New shared per-tick state as a specs Resource, threaded through execution-data structs — not a global, not a side channel (§1).
- [ ] A `specs::Entity` (or any recyclable index) in a persisted or cross-tick struct? Rejected — stable IDs only (§1).
- [ ] Generic trait/dispatch surface gaining an `is_<specific-thing>` method? Rejected — use the AGENTS.md §8 ladder (§2).
- [ ] New segment or shared-namespace id? A registry constant behind the compile-time uniqueness assert — never locally defined (§2).
- [ ] Any failure path silent? Every swallowed error logs loudly AND counts in seg-57 (§3).
- [ ] Unbounded loop/search/recursion? It draws from a budget with defined degradation (§4).
- [ ] Serialized-shape change? Reset-anytime applies — but the reset must be LOUD, the change carries its breaking-change label, and in-flight multi-tick state gets a fingerprint bump (§5).
- [ ] Extracted pure logic ships kernel/pin tests in the same commit; strategy tweaks ship none (§6).
- [ ] Quantitative/mechanics claim? Engine-cited or independently recomputed — including the reviewer's own arithmetic (§7, §8).
- [ ] Hand-rolled encoding, shell script for a durable tool, repo-relative default path, env-var config? All rejected (§9).
- [ ] Commit: no AI attribution, leaf-first submodules, task ID when one exists; builds warning-free on both lanes (§10).

## 1. State & ownership

- **EP-1.1 — No mutable `static`s or `thread_local!`s in bot code.** The sanctioned exceptions, exhaustively: (**EP-1.1a**) the **single top-level game-environment instance** (`game_loop.rs` `ENVIRONMENT`) — the composition root that owns the `World` across JS→wasm calls; it is ownership, not shared state (never read below `tick()`), and there must be exactly one; (**EP-1.1b**) the platform trio — `#[global_allocator]`, the panic hook, the logger; (**EP-1.1c**) a function-local once-per-VM log latch (logging only, never control flow); (**EP-1.1d**) feature-gated profiling/dev instrumentation in dependency crates (`screeps-timing`'s `TRACE`, `profile` builds only). Everything else is a Resource or a passed value. *Why:* statics are process-wide — they cross-contaminate parallel host tests (`cargo test` runs threads; a tier `refresh()` in test A flips test B's assertion), leave shadow-dispatch comparison no way to diff two dispatches short of snapshot/reset choreography (a Resource is simply two instances), and block Inc-6 record/replay (replayed ticks need per-instance injectable state). The CAS/Mutex ceremony they force is dead weight on single-threaded wasm. — Operator directive 2026-06-11; full inventory + migration plan in [`../reviews/statics-and-pathfinding-review.md`](../reviews/statics-and-pathfinding-review.md).
- **EP-1.2 — Shared per-tick state is a specs Resource, fetched via `SystemData`, threaded into job/mission/operation code through the execution-data structs.** The in-tree precedent is `transfer_queue: Write<TransferQueue>` → `&mut` field on `JobExecutionRuntimeData` → every job sees it with zero per-impl edits. Mutable handles ride the `&mut` runtime-data side, never the shared `&` system-data side. — `jobs/jobsystem.rs`; feasibility analysis in the statics review.
- **EP-1.3 — State as a field on the system struct only when exactly one system reads AND writes it.** If a second system (telemetry counts) or pre-dispatch code (`tick_start`) needs it, it's a Resource. Nothing in the current inventory qualifies for system-struct state. — statics review §3.
- **EP-1.4 — JS-side `Memory.*` is a boundary, not a convenience global.** Rust state goes there only when it must survive a VM halt (`_metrics.vm_starts`, `aborted_ticks` — the heap dies, Memory doesn't) or be readable before the world exists (`_features.reset.*`). Anything else in Memory is hidden global state with extra steps. — P1.C2; `metrics.rs`, `features.rs`.
- **EP-1.5 — Review smells:** an `AtomicU32`/`Mutex` static on the wasm target (concurrency ceremony standing in for ownership); a static whose comment says "reset at tick start" (that's a Resource wearing a trench coat — `tick_start(&mut World)` is already the writer seam); a free function reading ambient state that every caller could have passed in.
- **EP-1.6 — Until a static is migrated, it carries a written justification at its definition** (per-tick reset point, Memory-backed twin, or rebuild-on-reset path) and its mutating tests stay serialized (one mutating `#[test]` per module). The one current gap — `globals.rs` `USERNAME`, which carries no justification — is slated for migration (review M6); don't add a second. — code convention.
- **EP-1.7 — Durable or cross-tick references are keyed by stable IDs** (`RoomName`, `ObjectId`, minted monotonic ids) — never recyclable `specs::Entity` indices; id→Entity maps rebuild each tick from live state. Review smell: an `Entity` field in any persisted or cross-tick struct. *Incident:* a raw `entity.id()` u32 re-attached to whatever lived at that slot after a VM reset, silently aliasing another squad's orders (IBEX-002b) — and the bug class re-entered twice at design review (`BoostQueue.requester: Entity`; graph keys). — ADR 0001; ADR 0010 §4, 0009 D3.1.

## 2. Seams, abstractions & ownership

- **EP-2.1 — One change, one named frozen seam; callers never move** (strangler-fig). Each increment is independently verifiable and reversible behind its seam (`serialize_world`, Job/Mission traits, `run_systems`, the matcher snapshot seam…). Never break the running bot mid-increment. — rewrite-plan §1/§3; every ADR migration path.
- **EP-2.2 — New plumbing lands at behavior parity first; behavior changes turn on only after parity is verified.** Two failure sources are never debugged at once (scheduler seam shipped same-order/no-shed; screeps-pack shipped byte-identical before deploy.js died). — ADR 0005; P0.A13.
- **EP-2.3 — Seams grow consumer-by-consumer; no speculative scaffolding.** "Add a method when a consumer migrates, never speculatively" (`gameview.rs` growth rule); create the testkit crate when the first fixture needs it. A premature recorder/abstraction records a thin slice — false confidence. The one sanctioned exception is irreversible-if-missed *data capture* (EP-6.12): capture may start before its consumer exists; runners and abstractions still wait for theirs. — P1.A6; P0.B2.
- **EP-2.4 — Keep shared abstractions generic — no usage-specific bleed.** The escalation ladder lives in AGENTS.md §8: (1) keep the info with its owner, (2) concrete accessor + `match` at the strategy-owning caller, (3) opt-in capability trait — never widen the base trait. Review smell: `is_<specific-thing>()` on a generic surface. — operator directive; AGENTS.md §8.
- **EP-2.5 — Mechanism/policy split, genericity verified not asserted.** Generic toolkit crates carry zero project-specific strings (server-kit verified by synthetic marker); policy lives in the policy crate (ibex-eval). Tool crates are designed for extraction from day one: no workspace-internal deps, no repo-relative paths, README written for external users. — P0.A14, D-1.
- **EP-2.6 — One implementation per concern, not N.** Duplicated infra consolidates into a shared crate/module (one REST client; one room-scoring implementation), with bit-identical behavior verified at consolidation (43/43 rankings unchanged). Supplanted code is deleted (zero call sites verified by search), not flagged off; dead-but-planned code sits on an explicit KEEP list with an ADR owner. — P0.A12/P7; phase-0 §3.
- **EP-2.7 — One writer per shared datum; every long-running activity has exactly one lifecycle owner and a definite terminator.** Multi-writer smear ("everyone's job, therefore no one's") is the root pattern behind the combat lifecycle failures. Liveness/cohesion conditions are measured invariants with enforcement (force-abort after N bad ticks) — never gates that relax themselves under failure (the `squad_is_cohesive` drop-the-requirement-after-15-ticks anti-pattern hid the failure it should have terminated). — ADR 0008/0014; IBEX-001.
- **EP-2.8 — Shared-namespace allocations get a single authoritative registry with a compile-time twin.** Segment IDs: named constants in one module, whole-table uniqueness `const` assert, subsystems import — never locally define. An ADR allocating a namespace entry without a registry row is incomplete. *Incident:* the seg-55 cost-matrix wipe (Critical, IBEX-013) was exactly an unregistered allocation. — ADR 0002; `segments.rs`.
- **EP-2.9 — Make invalid states unrepresentable where cheap:** capability-split enums (withdraw-capable vs deposit-capable `TransferTarget`), shared contract constants exported from one crate (the foreman container/link ranges), single-source macro generation for lists that must not drift (`for_each_system!`). — ADR 0007 §5; ADR 0009 D2; `game_loop.rs`.
- **EP-2.10 — Interim scaffolding is labelled and must not accrete callers.** Temporary bridges become permanent unless fenced (the 0001 A1 generation-handle rule). — ADR 0001.

## 3. Failure semantics — loud over silent

- **EP-3.1 — Every reset has a logged cause; a decode failure is a loud reject-and-reset, never a silent empty world.** *Incident:* Field Report D — silent decode→empty presented for months as "spontaneous colony wipes". The previously-silent path is now loud + counted (P1.A1). — rewrite-plan §1 non-negotiable 2.
- **EP-3.2 — Swallowed errors count, not just log.** Every fault path gets an always-on counter in the seg-57 metrics block (deser failures, panics caught, move failures, dropped visuals) so it trends instead of living in grep. Telemetry distinguishes *shed* (deliberate) from *aborted* (caught) — otherwise shedding reads as faults and faults hide as shedding. — ADR 0005; P1 metrics.
- **EP-3.3 — No silent no-ops on write paths.** A helper that can't complete a write creates the missing intermediates or fails loudly. *Incident:* `memory_helper::path_set` silently dropped writes through missing intermediate keys — found live when `vm_starts` vanished. — P1.C2 bonus find.
- **EP-3.4 — Hot path: log-and-continue, never unwrap/panic; per-item isolation in loops** (one room's spawn failure must not kill the loop). Known-risky boundaries get narrowed to `Result` at the source; the tick-level containment boundary is the backstop, not the primary defense. — AGENTS.md §8; ADR 0005.
- **EP-3.5 — Invalid input to an unsheddable output path is dropped-with-telemetry, never propagated** (non-finite visual coords corrupt the target's whole payload; cap + count + warn-once). Repeating warnings get a once-per-VM latch or an aggregated count — log spam is a defect. — P1.C6; `visualize.rs`.
- **EP-3.6 — Assert invariants at the source, governed by the ADR 0015 §2 build-profile matrix:** panic on host/sim, log-and-count on wasm, compiled out of MMO release — never an asserting validator in any deployed wasm artifact (under `panic="abort"` it costs the whole tick). The current in-tree form is `debug_assert!` + a `#[cfg(debug_assertions)] #[should_panic]` pin (acceptable while no debug-wasm artifact is deployed); the `ibex_invariant!` macro is the planned mechanism — adopt it when it lands. — IBEX-046 pattern ×3; ADR 0015 §2.
- **EP-3.7 — Don't advance the epoch/tick marker until persistence succeeded.** A mid-pass abort must roll back to the last good snapshot, not continue on half-applied heap state. — ADR 0005; `game_loop.rs` env.tick ordering.
- **EP-3.8 — Transient faults park work (`Wait`), they don't tear it down** — a one-tick visibility blip must not delete a long-running campaign. Fail toward the safe posture on state loss (no surprise offense after a reset). — ADR 0003 A6; ADR 0014.

## 4. CPU & budget discipline

- **EP-4.1 — Every unbounded or expensive operation draws from a budget, and budgets charge a shared pool.** Per-call-site caps alone are insufficient — N capped sites × per-search caps is the aggregate leak class that killed colonies (Field Report C, extinction-class). `find_route` is *charged*, not merely gated. — ADR 0004; P1.B1/B4.
- **EP-4.2 — Every budget defines its degradation where the grant is issued:** exhausted pool ⇒ the caller's existing "no path" semantics; Critical tier ⇒ serve-stale over recompute ("a stale route is strictly better than a fabricated answer"); missing entries always compute. — `pathbudget.rs`, `economy.rs`.
- **EP-4.3 — The never-shed set is tiny and survival-gated** (defense, spawn, haul, movement, `serialize_world`); admission bar is survival-criticality, not importance. Changes amend ADR 0004. **Telemetry never sheds** — you must be able to see the degraded state. A minimum-progress floor (`MIN_PATHFIND_OPS`) means nothing ever fully freezes. — ADR 0004; P1.C5.
- **EP-4.4 — Shed re-decision, never committed work** (ride existing tickets at Critical; freeze re-planning, not the cheap status poll). Tier decisions use **trend, not just level** — the death-spiral alarm fires on slope. — ADR 0004/0007/0011.
- **EP-4.5 — Anything that auto-restarts gets bounded attempts + exponential backoff + a loud terminal state.** *Incident:* planner restart-thrash (IBEX-037) burned the whole planning budget forever, producing nothing. — ADR 0009 D1; P1.D3.
- **EP-4.6 — Tuning numbers are measured, not asserted:** cadence/threshold changes ship with before/after evidence; calibratable *bot-side* thresholds are named constants (not runtime config) so calibration lands as a reviewed diff (harness *gate* numbers are the opposite case — one reviewed config, EP-6.7); CPU accounting includes intent cost (0.2 each, engine-pinned). — P1.B6; `cpugovernor.rs`; engine-mechanics.
- **EP-4.7 — For emergent, non-reproducible failure classes, don't chase a repro — make the defensive property first-class and validate with synthetic pressure** (the CPU burner, not a death-spiral reproduction). — review §Field Report C; P1.A5.
- **EP-4.8 — Persist/warm a cache across resets when recomputing it on the post-reset tick is itself the failure mode.** The post-reset tick is the most starved tick; ephemeral caches re-armed the route storm precisely there (seg-55). This is the scoped exception to EP-5.4. — ADR 0004 §3; ADR 0002.

## 5. Serialization & persistence

- **EP-5.1 — Rewrite-period policy (operator, 2026-06-10): reset-anytime.** Serialized state may drop at any increment; build NO migration paths, version bridging, or reset choreography until the holistic serialization pass at the Inc-7 entry gate ("versioning formats while the data structures are still churning buys migration debt, not safety"). **Loudness is the entire requirement** (EP-3.1). This supersedes AGENTS.md §6's back-compat rules until Inc 7. — rewrite-plan §0.
- **EP-5.2 — Still in force: never break the *currently-running* bot mid-increment;** every change carries its breaking-change label (None / Behavioral / Memory-format); no MMO deploy between a format break and an accepted state-drop window. — rewrite-plan §3; phase-0 §0.
- **EP-5.3 — When a resumable process's semantics change, force invalidation of its in-flight persisted state via a fingerprint/version bump** — never let old in-flight state complete under new code (`controller_infra_v2` rename invalidating seg-60 planner state). — ADR 0009 D2.
- **EP-5.4 — Don't persist what is derivable or re-assertable** (demands re-assert cheaply; in-flight spawns recover from game state; mark derivable fields `#[serde(skip)]`) — *unless* recomputing it on the post-reset tick is itself the failure mode (EP-4.8). Every persisted byte is segment pressure plus a future format-break surface. — ADR 0011/0007/0013.
- **EP-5.5 — Telemetry/metric schemas are versioned from day one with additive-evolution rules pinned by tests;** new fields bump the version; the writer and reader share the schema type through a host-compilable crate. (Exempt from EP-5.1: the reader is the harness, which lives outside the resettable world — telemetry versioning is reader-compat, not state migration.) — `screeps-ibex-metrics`; ADR 0006; rewrite-plan §3.1(5).

## 6. Testing & validation

- **EP-6.1 — Kernel-vs-shell:** extracted pure decision logic ships kernel/fixture tests *in the same commit* — the 0015 bar is unit coverage **plus at least one property/relation test** per stable kernel; strategy/tuning tweaks ship **no** new tests (scenario gates + snapshots pin them). "Is this module shell? Then why does this PR add a mock?" The deliberately-untested list is explicit — silence is how test bureaucracy creeps in. — ADR 0015 §3.
- **EP-6.2 — Build decision logic pure-by-design:** DTO inputs, no JS-bound game-API types below a seam line (allowed: JS-free value types — `Position`, `RoomName`, `LocalCostMatrix` — and opaque ids; banned: `Creep`/`Room`/`RoomPosition`, anything needing `.resolve()` or `game::*`); JS-touching impls quarantined in feature-gated leaf modules. Testability is cheap at design time and an M-L refactor afterward. — ADR 0015 §4; component-test-plans.
- **EP-6.3 — Pin style:** `#[cfg(test)] mod tests` is the last item in the file; pins state the incident/decision that makes the behavior load-bearing ("The raise itself is load-bearing (IBEX-021: all three were 1)", `war.rs:1495`); fixture/matrix tests iterate the input space with labeled asserts; formula pins cite the mis-estimate they replaced (renew: "the pre-D4 estimate charged 400 — 10x"); REFUTED-finding guards pin correct-but-suspicious code so nobody "fixes" it into a bug (the spawn comparator). — tree-wide convention.
- **EP-6.4 — Behavior-preserving conversions prove it:** a parity test against the legacy formula ("behavior-preserving by construction" + `matches_legacy_formula`), a parity instrument for stream-shaped changes (intent digest, shadow-dispatch), or byte-compare for artifacts (screeps-pack vs deploy.js). Determinism fixes (sorted iteration) land *before* any parity recording. — P1.B3/C3; P0.A13; ADR 0015.
- **EP-6.5 — Test doubles are built into the production type behind `#[cfg(test)]`, unconstructible in production** (`MemoryArbiter::test_double()`, `FixedGameView`); game-API access goes through object-safe trait seams so doubles swap behind `&dyn`. — P1.A6/A7.
- **EP-6.6 — Both lanes green before done** (host `cargo test` + `cargo clippy-wasm`); seams land with their draft contract files; warning-free is the standing bar across workspace AND tool crates — `#[allow(dead_code)]` is targeted per-item for phase scaffolding, never blanket-tolerated warnings. — phase-1 §2.1; fa346d0.
- **EP-6.7 — Gates: hard zeros for single runs (zero panics/deser, ticks observed > 0) — single-run metric thresholds are flake generators.** Statistical gates are N-seeded paired diffs vs a stored (scenario, seed, SHA) baseline, never absolute thresholds ("tautologically green at birth"); gate numbers live in one reviewed config, "a gate may not cite a literal". CI retries are forbidden — L0–L4 flakes are real defects, fix-or-delete. — phase-0 C6; ADR 0015 §3; F19.
- **EP-6.8 — Comparisons use plan-loss accounting: entities that *drop out* of the result set are regressions.** Intersection-only diffs hide them; recompute quality metrics from ground truth (terrain + final structures), never from the system's self-reported scores. *Incident:* the first bench comparison diffed common rooms only and reported "zero regressions" while E11N25 had been bricked. — ADR 0009 D2 standing gate.
- **EP-6.9 — Run hygiene:** smoke per batch (not per change — amortize the expensive lane, keep pins/clippy per-change); **never edit bot sources while a smoke runs** (deploy builds from the working tree — the run silently measures nothing); runs land under `runs/` keyed (scenario, git SHA). — phase-0 §4; validation cadence.
- **EP-6.10 — Know your fault probe; prove containment by inducing the failure.** The reset probe is the state-preserving `Memory._features.reset.*` path — `system.resetAllData()` is a WORLD WIPE and can never be the probe. Containment/recovery paths are validated live with loud-accounting assertions (panic probe → counters + survival); structural survival changes get a live E2E proof, not just unit tests. — phase-1 §1.2 trap note; P1.C2.
- **EP-6.11 — Optimizations are validated against the constrained/worst case, not the nominal case** (tight-layout rooms for placement; "ensure this won't cause congestion"). — operator directive, D2 work.
- **EP-6.12 — The harness never lies:** infra failures are never reported as bot failures; the harness has its own known-good self-test; capture starts before its consumer exists (you can't retroactively capture old-format snapshots — the EP-2.3 exception). — component-test-plans §15.
- **EP-6.13 — Determinism is a prerequisite of decision code:** no `HashMap`/`HashSet` iteration order may reach a decision or an emitted ordering — sort or use `BTreeMap` at the boundary. *Incident:* spawn fulfilment picked its room by `HashSet` order. Review smell: `.iter()` on a hash collection feeding a choice. — ADR 0015 §3; ADR 0011.
- **EP-6.14 — One artifact per gate, one owner:** a validation promised in N documents is built once; new ADRs/plans must reference an existing harness artifact before minting a new one (the sweep found 17 duplicate-gate clusters — duplicated gates drift and none is authoritative). — ADR 0015 §1; component-test-plans.

## 7. Engine & external ground truth

- **EP-7.1 — Check [`engine-mechanics.md`](../references/engine-mechanics.md) before guessing mechanics; load-bearing numbers are engine-cited, not folklore.** *Incidents:* "boost while spawning" is engine-impossible; renew cost was mis-modeled ~10×; IBEX-050's premise was FALSE against engine source — the fix became a deletion. Workarounds are re-verified against current upstream before being kept. — design README; P1.D4; phase-0 D6.
- **EP-7.2 — Do the arithmetic before deciding;** alternatives get rejected by math, not taste (the 20×ATTACK cap exists because 25 out-reflected its own healer, 375 vs 300). Record folklore corrections as numbered facts and never re-litigate them. — ADR 0010/0013; gap-analysis §2.
- **EP-7.3 — Read engine values at runtime where they can vary; replace guessed constants with measured values plus a sanity band that alarms when the measurement leaves it.** — ADR 0011/0014.
- **EP-7.4 — Validate heuristics against large ground-truth datasets and record their limits** (claimability vs 77,776-room dumps: matches the `bus` flag exactly, recorded necessary-not-sufficient with the counterexample pinned). — MMO wave.
- **EP-7.5 — Gate on the precise distinguishing property, not a proxy** (the `is_official()` proxy was wrong — token-auth private servers are legitimately shardless; the distinguishing property was the screeps.com host); re-apply derived predicates when reading stale caches. — prospector fix 0452c66.

## 8. Review methodology

- **EP-8.1 — Adversarial verification before publication:** every finding is confirmed/located/adjusted/refuted against current code with file:line evidence; refuted findings are never presented as confirmed. Operator-observed production failures outrank static analysis. Review phases are strictly read-only — fixes are proposed as reviewable doc entries, never applied inline. — review prompt; report header; operator directive.
- **EP-8.2 — REFUTED seeds are immunized:** recorded explicitly ("do NOT re-flag"), with a lock-in pin + clarifying comment on the correct-but-suspicious code. The real risk is "fixing" correct code into a bug. — report INFO exemplar; `spawnsystem.rs` pin.
- **EP-8.3 — Independently recompute every quantitative claim — including the critic's.** *Incident:* the adversarial reviewer's "8 slots / link outside" arithmetic was wrong, and so was the first in-house count; the true optimum (7 slots, in-coverage link, 31>30) emerged only on recomputation. Verification is symmetric. — D2 work-slot session.
- **EP-8.4 — Severity weighs survival/gameplay impact, not style;** in-flux areas get design-level guidance, not line nits; breadth-first flag-and-track beats exhaustive diagnosis during reviews. — review prompt §7/§10.
- **EP-8.5 — A fix whose honest validation instrument doesn't exist yet is deferred until the instrument lands** (IBEX-029 waited for the intent differ). "Behavior-preserving" is a claim that needs its measuring device. — phase-0 D4.
- **EP-8.6 — Adversarial review applies to plans and docs too:** plans get the same confirm/refute pass as code, and accepted corrections are recorded inline at the corrected spot ("per adversarial review") so the document shows its own audit trail. It caught the E11N25 comparison flaw and the recorder chicken-and-egg. — component-test-plans.

## 9. Dependencies, tooling & configuration

- **EP-9.1 — Never hand-roll well-solved encodings or protocols** (base64/hex/json → the standard, widely-audited crate). *Incident:* a hand-rolled base64 encoder + alphabet table in rest-api (P0.A12.1). Dependency bar: widely used, industry-regarded crates (tokio, bollard). — operator directive.
- **EP-9.2 — Prefer Rust over Node/JS; programmatic over shell, for anything durable.** Durable tooling is portable library-backed code (bollard, not shelled `docker`); one-off interactive commands are fine. Reuse `screeps-game-api` types and serialization rather than re-parsing game data. — memory: prefer-rust-over-node, prefer-programmatic.
- **EP-9.3 — Tool binaries that must match a lockfile are resolved from the lockfile, never static-pinned** (wasm-bindgen CLI — version skew silently corrupts output). — P0.A11/A13.
- **EP-9.4 — No repo-relative default paths in tool crates;** defaults must work for any user's checkout (the documented exception: the sibling-convention `../.screeps.yaml` family). Config via files, not env vars (Windows-hostile, brittle). Design config for the dominant use case — no speculative multi-tenant complexity. — P0.P8; operator directives.
- **EP-9.5 — Secrets are safe by construction, not by call-site discipline:** `SecretString` at the config boundary so `Debug` redacts; a pin test asserts no secret in formatted output; known limits are *named* (the CLI payload path) and covered by manual sweep; never vendor files containing live secrets (the launcher `config.yml` steamKey); secrets never enter `runs/` artifacts. — P0.A7.
- **EP-9.6 — Silent long waits are surfaced, and throttling engages on evidence, never assumption** (a backoff invisible in-terminal looks like a hang). — MMO wave.
- **EP-9.7 — Human commands and automation share one code path** ("no parallel code path") — the operator exercising what automation uses is how the pack-defaults bug was found. — P0.A8.

## 10. Process, git & hygiene

- **EP-10.1 — Living docs:** phase docs separate Design (end state, changes only on decision change) from Execution (status log newest-first, cold-resume instructions, task IDs `P<n>.<ws><n>` in every commit, status updated in the same commit). Every task gets an honest disposition — done / deferred-by-decision / rescoped / obsolete — never silently dropped. Phases close with a per-criterion exit audit (verdicts MET/PARTIAL/pending with evidence pointers, never rounded up) and an explicit carried-forward list. — phase-0/1 conventions.
- **EP-10.2 — Confidence-tiered planning:** near increments planned in detail, far ones deliberately under-planned and re-planned at their gate; speculative late items are an evidence-gated menu, not a queue. — rewrite-plan §0.
- **EP-10.3 — Commits: no AI attribution** (no `Co-Authored-By`/"Generated with" trailers — project-wide, no exceptions); detailed imperative messages; checkpoint commits at milestones; **leaf-first submodule ordering** (submodule repos, then the superproject pointer); `screeps-game-api` changes land via upstream PR, never direct pushes; never commit or revert the operator's in-flight uncommitted work. — memory: no-ai-commit-attribution; AGENTS.md §9.
